How to make a website

I made this website for free using GitHub's domain hosting and Nicepage's website builder. Here's how to do it.

Step 1: Make a GitHub repository

To make a website the first step is to have a space to host your website. ​ A convenient and cost-effective option is to establish a GitHub repository named <GitHub username>.github.io, where you can host your website. For instance, the website you are currently exploring resides within a repository titled rowina-n.github.io.

I​f you haven't already, creating a GitHub account is a simple process found here.

GitHub also have their own instructions on hosting a website in a repository here. 

Step 2: Design your website

This website was designed using Nicepage's website builder. It is free (there is a premium version, I haven't personally used it) and has a desktop app. It is easy to use. Make a page, add blocks (text, images etc.) and away you go. I would reccomend having a design and colour scheme in mind before you start. Also, as you are building I would use the available grid layouts rather than dragging and dropping things, that way your website scales nicely to different devices. 

Once your website is looking ready to go live (you change it easily at any time) you need to have a html file. This can be done by pressing the export button in Nicepage and choosing a html file. The html file option has a premium logo on it but do not worry! It will tell you that html export is a premium feature but will let you do it anyway, just click through the menus.

Nicepage only allows one site on the free version. If you have already made a website, or would prefer to do things the hard way, you can download a html5 template here and alter it to build your website. There are many free templates here, these include the css file, which is like a style sheet. Making a website this way is much more of a steep learning curve. I made two websites using this method. When it comes to actually writing up the website, I would reccomend using VS code. It can show you a live render of your website as you code it.

Once you have all the html and relevant files for your website the last step is to push to GitHub! 

Step 3: Push .html file to GitHub

Open the folder that the website is contained in and push it to the repository made in step 1. If you don't know how to push to a remote GitHub repository instructions can be found here. 

GitHub should automatically publish your website now! Go to Settings > Pages and there should be a button that links to your site. If things aren't working, try using the GitHub guide here. 

Now you've published a website! It is live at <github username>.github.io. If you want to use a custom domain, you'll have to google how to do it. Your website can be changed as often as you like just make the edits, save to html, push to git hub and boom! new website.
