Getting started with PSRchive

This is a guide on how to produce time of arrival estimates from archive files. First, install PSRchive. 

To install PSRchive you want to go here: https://psrchive.sourceforge.net/installation.shtml 
I personally prefer the bash instructions. 
There is a suggestion to install using conda but that didn't work for me. I installed it using the instructions here: https://psrchive.sourceforge.net/current/ 
You then need to build the files you cloned as follows https://psrchive.sourceforge.net/current/build.shtml 

Once installed, you can check that the installation worked by running pat -h

Things you need 

There are a number of files you need to make time of arrival estimates. These are listed below.

Archive Files

You need a directory with archive files in it. An example of these kind of files can be found here: https://data.csiro.au/domain/atnf?redirected=true 

These contain the raw flux data from a telescope across time, frequency and polariasation. 

Meta File

I find it easiest to list all my archive files in a meta file so that PSRchive can find them. This should be a text file with the names of all archive files. This can easily be created by changing into the directory containing the archive files, then running ls > archives.meta in the terminal.

Pulse Template

You need to create a pulse profile template for PSRchive to fit to the individual pulses/epochs. You can make these yourself with PSRchive. This is the usual command psradd -o template.average -M archives.meta

Make sure to plot it once you've made a template to make sure it makes sense. The pulses need to be aligned to do this. There are examples of how to do this in the PSRchive tutorial below. You can use something like pam -r 0.5 <archives> -e <file extension for turned archives> but this turns all the pulses by half.

You can also smooth it with psrsmooth -W template.average

 The file output by this command will be template.average.sm. There are also ways to make templates in python using Pulse Portraiture.

Making time of arrivals

Making a time of arrival file is now as easy as runing
pat -F -T -f tempo2 -s template.average -M archives.meta > pulsar.tim

There are a number of settings included in this terminal command, these are outlined below:
pat is the pulsar timing package in PSRchive
-F frequency scrunches the data 
-T time scrunches the data 
-f is for the output format, I use tempo2 (more about tempo2 later) 
-s is pointing to the template file
-M tells PSRchive you want to use a metafile
> saves the output to a text file

More instructions can be found here: https://psrchive.sourceforge.net/manuals/pat/, or by running pat -h

You should now have a .tim file with time of arrival estimates for your archive files!
